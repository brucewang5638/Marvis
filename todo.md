### 你要的是一种“有连续性的大脑”

换句话说，你不是做一个对话框，而是做一个“会长出生活痕迹的系统”：

1. **长期记忆模块**：聊天时，提到了重要信息（你喜欢什么、你在做什么项目、偏好如何……）它能自动提取并存储。这不是简单 chat history，而是真正结构化数据。
2. **TODO / 任务模块**：你随口说“帮我记一下，明天要打包 docker 镜像”——应用自动捕捉并记录到任务系统中。
3. **Chat 模块**：贾维斯式的自然交互，随时召唤，并能引用你的记忆和任务。
4. **计划书自动生成**：有了聊天记录、任务、偏好，随时能导出「产品计划书 / 大纲」。

一个“轻量级私人助手”，但带一点魔力。

### 功能视角：一个最小可用“MVP贾维斯”

我帮你缩成第一版可以做出来的能力，不会太复杂，但足够「能跑」。

#### 1）聊天界面

你用 React / Vue / Electron 都可以。你本来在做桌面应用，其实很适合内嵌这个模块。

#### 2）长期记忆存储（重点）

这个不是大模型自动“记住”，而是你设计一个记忆的 schema。

**最小版可以这样分类：**

* profile（你的个人喜好、习惯）
* preferences（偏好的工具、写代码风格、你正在学 Kotlin…）
* active projects（你开启的项目）
* todos（任务）
* insights（聊天中提到的长期有价值的信息）

**底层存储方式可以很简单：**

SQLite
或
JSON 文件（更暴力但 MVP 非常够用）

**示例结构（SQLite）：**

```
memory(id, type, content, created_at)
todo(id, content, status, created_at, updated_at)
```

模型回答时，把 memory 表的内容作为“context”提供过去。

### 技术路线：最简单、但具未来扩展性的

你现在的技术栈是：

* Kotlin 代替 Java
* TS 代替 JS
* 会写后端 / 会写 Electron 桌面工具
* 会用大模型 API（例如 OpenAI、智谱、DeepSeek 等）

基于这些，你可以很轻松构建：

1. **后端（K tor or Spring Boot + SQLite）**

   * `/chat` 接口负责模型调用
   * `/memory` 接口负责保存、查询记忆
   * `/todo` 接口处理任务

2. **前端（Vue + Electron）**

   * 聊天窗口
   * 任务页面
   * 记忆页面（调试用，正式版隐藏）

3. **大模型调用（OpenAI / DeepSeek / 本地 vLLM）**

   * 每次回答前，都先从 memory 中查相关记录
   * 把它打包作为 system prompt 输入

### 关键价值：让 AI 自动决定“什么需要记住”

这才是有趣的灵魂。

给你一个简单的“记忆过滤 prompt”逻辑：

```
你是一段辅助记忆程序。
用户每句对话都是 user_content。
你需要判断：是否包含需要长期存储的新信息？
如果需要，请返回一个 JSON：
{ "should_write_memory": true, "memory_type": "...", "memory_content": "..." }
否则返回：
{ "should_write_memory": false }
```

这样你的应用里就能自动抽取记忆，不需要你手动点。

---

### TODO 捕捉（超级自然语言）

只要用户说：

“帮我记一下：明天更新一下数据库检查工具的 Redis 模块。”

模型会自动输出：

```
{ "todo": "明天更新数据库检查工具的 Redis 模块" }
```

然后系统写到 todo 表。

### 计划书的自动生成（非常酷）

有了 memory + todo + 聊天内容，你可以在 UI 放一个按钮：

【一键生成产品计划书】

模型 prompt 示例：

```
使用 memory 表、todo 列表、近期聊天记录，自动总结出：
- 产品愿景
- 核心功能
- 技术架构
- MVP 范围
- 后续版本路线图
```

这就能产出一个结构清晰又贴合你的实际开发进度的计划书。
